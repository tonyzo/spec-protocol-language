# ADR-005: 量纲校验保留 + 参数表分区组织

## 状态
已确认（2026-06-29）

## 决策

### 1. 量纲校验保留
- 每个参数与限值都携带 `unit` 字段。
- 断言求值**前**先校验两侧 unit 一致性（如 `δ.unit == δ_min.unit`）。
- **不一致则报错（Err），不静默换算**。
- 校验范围：仅一致性检查，不做 mm↔m 自动转换（避免隐式行为）。

### 2. 参数表分区组织
输入参数表（JSON/YAML）分为两个区：
```yaml
parameters:
  delta: { value: 16, unit: mm }
  sigma_theta: { value: 120, unit: MPa }
limits:
  delta_min: { value: 12, unit: mm }
  sigma_allow: { value: 163, unit: MPa }
```
- 断言跨区引用：`parameters.delta >= limits.delta_min`
- 规则的 assertion 表达式中用 `param.xxx` / `limit.xxx` 前缀区分来源。

## 理由
- 量纲校验虽不做换算，但能防止计算软件输出单位不一致导致的误判（GB150 docx 约束强调量纲）。
- 分区组织让「值」与「限值」职责清晰，断言语义明确。
