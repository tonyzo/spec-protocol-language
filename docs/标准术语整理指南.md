# 标准术语整理指南

> 本指南描述如何将工程标准(PDF/Word/Markdown)整理为SPEC Protocol可用的术语表和参数映射。
>
> **适用场景**:
> - 新标准接入(首次)
> - 标准更新(换版)
> - 定期批量整理(全量校验)
>
> **工具**: `term_extractor` (独立脚本,待开发)

---

## 1. 整理流程概览

```
标准原文(PDF/Word)
    │
    ├─ 1. 术语提取 ───────────────────► 术语定义表(草稿)
    │                                      ↓
    │                                 人工校对确认
    │                                      ↓
    ├─ 2. 参数映射 ───────────────────► param_mapping.json
    │                                      ↓
    │                                 校验一致性
    │                                      ↓
    ├─ 3. 文档生成 ───────────────────► {标准号}-术语定义与概念映射.md
    ├─ 4. 规则骨架 ───────────────────► rules/library/{标准号}/*.yaml
    └─ 5. 关系分析 ───────────────────► {标准号}-条款引用关系.md
```

## 2. 步骤1: 术语提取

### 2.1 识别术语定义章节

标准文档中通常在以下位置包含术语定义:

| 标准 | 术语章节 | 示例 |
|-----|---------|------|
| GB150.1 | 第3章 术语和定义 | 3.1.3 工作压力, 3.1.4 设计压力 |
| ASME VIII | UG-20 Definitions | Design Pressure, Design Temperature |
| EN 13445 | Clause 3 Terms | Design pressure, Maximum allowable stress |

**操作**:
1. 打开标准文档,定位到"术语和定义"章节
2. 逐条提取术语(术语名称 + 定义 + 条款号)
3. 标记每个术语的数据类型(数值/字符串/布尔)

### 2.2 术语分类

提取的术语按SPEC Protocol三分区分类:

| 分区 | 含义 | 判断标准 | 示例 |
|-----|------|---------|------|
| **parameters** | 计算软件提供的计算值 | 需要数值计算得出 | sigma(应力), delta(厚度), design_pressure |
| **limits** | 判据边界值/阈值 | 标准规定的限值或查表结果 | delta_min(最小壁厚), sigma_allow(许用应力) |
| **attrs** | 分类属性(字符串) | 用于适用性筛选,非数值 | element_type, material_category, medium_toxicity |

**判断口诀**:
- 如果是"计算出来的值" → `parameter`
- 如果是"标准规定的限值" → `limit`
- 如果是"类型/状态/标志" → `attr`

### 2.3 术语模板

```markdown
| 序号 | 标准术语 | 术语定义 | SPEC映射 | 数据类型 | 单位 | 来源条款 | 计算软件提供 |
|-----|---------|---------|---------|---------|-----|---------|-------------|
| 1 | 设计压力 | 设定的容器顶部的最高压力 | design_pressure | parameter | MPa | 3.1.4 | ✅ |
| 2 | 许用应力 | 设计温度下材料的许用应力 | sigma_allow | limit | MPa | 5.2.2 | ✅ |
| 3 | 元件类型 | 压力容器的组成元件 | element_type | attr | - | 多处 | ✅ |
```

## 3. 步骤2: 参数映射

### 3.1 创建param_mapping.json

基于提取的术语表,创建JSON格式的映射文件:

```bash
# 模板位置: rules/library/{标准号}/param_mapping.json
# 示例: rules/library/GB150/param_mapping.json
```

**核心字段**:

```json
{
  "terms": [
    {
      "term_id": "pressure.design",          // 术语ID(英文点分)
      "standard_term": "设计压力",            // 标准原文术语
      "definition": "设定的容器顶部的最高压力", // 术语定义
      "spec_mapping": {
        "type": "parameter",                  // parameter/limit/attr
        "name": "design_pressure",            // SPEC参数名(蛇形命名)
        "unit": "MPa",                        // 单位(attrs为null)
        "data_type": "f64",                   // f64/string/bool
        "allowed_values": [...]               // 仅attr需要,枚举值
      },
      "source_clause": "3.1.4",              // 来源条款号
      "referenced_by": ["GB150.3-5.3"],      // 哪些规则引用此术语
      "related_terms": ["pressure.operating"], // 相关术语
      "confidence": "high",                   // high/medium/low
      "note": "可选,补充说明"
    }
  ]
}
```

### 3.2 命名规范

| 类型 | 规范 | 示例 |
|-----|------|------|
| **term_id** | 英文点分,分类.术语 | `pressure.design`, `thickness.calc`, `element.type` |
| **spec_mapping.name** | 蛇形命名(全小写+下划线) | `design_pressure`, `sigma_allow`, `delta_min` |
| **标准术语** | 保留标准原文 | "设计压力", "许用应力", "计算厚度" |

### 3.3 参数分组

按功能将参数分组,便于规则编写时快速查找:

```json
"parameter_groups": {
  "design": ["design_pressure", "design_temperature"],
  "geometry": ["D_i", "D_o", "delta", "delta_effective"],
  "stress": ["sigma", "sigma_allow", "sigma_theta"],
  "material": ["R_m", "R_eL", "E_t", "KV2"],
  "welding_ndt": ["phi", "ndt_ratio", "rt_defect_level"],
  "attrs": ["element_type", "material_category", "medium_toxicity"]
}
```

## 4. 步骤3: 条款引用关系分析

### 4.1 识别引用关系

标准条款之间存在引用关系,需要识别:

| 引用类型 | 示例 | 说明 |
|---------|------|------|
| **内部引用** | GB150.3-5.3 引用 GB150.2附录C | 设计计算引用许用应力表 |
| **跨部分引用** | GB150.4-10.3 引用 GB150.3设计压力 | 制造检验引用设计参数 |
| **公式引用** | 公式5-6 引用 公式5-1 | 多层圆筒公式引用单层公式 |
| **表格引用** | 表7-4 引用 7.6.6.4 | 封头半顶角限值 |

### 4.2 引用关系模板

```markdown
### GB150.3(设计) 内部引用关系

GB150.3-5.3 圆筒计算
    ├─ 引用 5.2.2(符号定义)
    ├─ 引用 GB150.2附录C([σ]^t许用应力)
    └─ 供 GB150.4-7.3(壁厚用于PWHT判定)
```

### 4.3 输出文档

生成 `docs/{标准号}-条款引用关系.md`,包含:
- 章节内部引用图
- 跨章节引用图
- 跨标准引用图

## 5. 步骤4: 文档生成

### 5.1 术语文档模板

生成 `docs/{标准号}-术语定义与概念映射.md`,包含以下章节:

1. **核心术语定义表** (按类别分组)
   - 压力相关
   - 温度相关
   - 厚度相关
   - 应力与材料
   - 几何参数
   - 焊接与NDT

2. **分类属性术语** (attrs)
   - 容器/元件类型
   - 材料属性
   - 制造工艺属性
   - 工况与环境属性

3. **标准条款引用关系图**
   - 内部引用
   - 跨标准引用

4. **参数依赖关系图**
   - 计算软件需提供的参数分组

5. **规则参数映射速查表**
   - 每条规则需要的parameters/limits/attrs

## 6. 步骤5: 规则YAML骨架生成

### 6.1 基于术语生成规则模板

根据param_mapping.json,自动生成功率YAML文件骨架:

```bash
term_extractor generate-rules \
  --mapping rules/library/GB150/param_mapping.json \
  --template rules/.templates/ \
  --output rules/library/GB150.3/
```

### 6.2 规则模板结构

```yaml
# {标准号}-{章节号}-{主题}.yaml
#
# 规则库文件：候选规则，尚未绑定到任何设计类型。

# ─── 规则 1：{审查内容} ({标准号} {条款号}) ───
# {判定逻辑描述}
- id: {标准号}-{条款号}.{描述}
  clause: "{条款号}"
  severity: {error|warning|info}
  applicability:
    type: {eq|or|and|exists}
    value:
      - {attr: element_type}
      - "{适用值}"
  assertion:
    type: {ge|le|eq|when|and|or}
    value:
      - {param: 参数名}
      - {limit: 限值名}
  message: "{错误提示,支持{actual}/{expected}占位符}"
```

### 6.3 参数名替换规则

工具自动将param_mapping.json中的映射应用到YAML:

```json
// param_mapping.json
{
  "standard_term": "设计压力",
  "spec_mapping": {
    "name": "design_pressure"
  }
}
```

↓ 自动生成 ↓

```yaml
# YAML规则
assertion:
  type: ge
  value:
    - {param: design_pressure}  # 自动替换
    - 0.1
```

## 7. 步骤6: 校验与一致性检查

### 7.1 自动校验规则

```bash
term_extractor validate \
  --rules rules/library/ \
  --check-consistency \
  --report docs/术语一致性报告.md
```

**检查项**:

| 检查项 | 规则 | 示例错误 |
|-------|------|---------|
| **参数引用** | YAML中的param必须在param_mapping.json中定义 | `{param: unknown_param}` 报错 |
| **单位一致性** | 比较运算两侧unit必须一致 | `sigma(MPa) <= sigma_allow(无unit)` 警告 |
| **术语完整性** | 每条规则的referenced_by必须更新 | 规则引用了术语但未更新映射表 |
| **命名规范** | spec_mapping.name必须是蛇形命名 | `DesignPressure` 报错,应为 `design_pressure` |

### 7.2 人工校对清单

- [ ] 所有术语的`confidence`字段标记完成
- [ ] 低置信度术语(medium/low)已人工复核
- [ ] `referenced_by`字段与规则YAML一致
- [ ] `cross_references`完整记录跨标准引用
- [ ] 参数分组合理,便于后续规则编写

## 8. 常见问题

### Q1: 如何处理同义词?

**示例**: GB150叫"设计压力",ASME叫"Design Pressure",EN叫"Design Pressure (p_d)"

**解决方案**:
```json
{
  "term_id": "pressure.design",
  "standard_term": "设计压力",
  "synonyms": ["Design Pressure", "Design Pressure (p_d)"],
  "spec_mapping": {
    "name": "design_pressure"  // 统一映射到SPEC参数名
  }
}
```

### Q2: 如何处理标准中没有明确定义的参数?

**示例**: 标准说"厚度应不小于计算厚度",但没有明确定义"计算厚度"术语

**解决方案**:
```json
{
  "term_id": "thickness.calc",
  "standard_term": "计算厚度",
  "definition": "按公式计算得到的厚度(推断,标准未明确定义)",
  "confidence": "medium",  // 标记为中等置信度
  "note": "标准未明确定义,根据公式5-1推断"
}
```

### Q3: 如何处理标准换版后的术语变更?

**解决方案**:
```bash
# 比对两个版本的param_mapping.json
term_extractor diff \
  --old rules/library/GB150-2011/param_mapping.json \
  --new rules/library/GB150-2024/param_mapping.json \
  --output docs/GB150-术语变更差异.md
```

输出差异报告:
```markdown
## GB150-2011 → GB150-2024 术语变更

### 新增术语
- `MAWP` (最高允许工作压力) - 2024版新增,2011版无此概念

### 删除术语
- (无)

### 术语定义变更
- `设计压力`: 2011版"设定的容器顶部最高压力" → 2024版增加"不低于工作压力"

### 参数映射变更
- `delta` 从 "设计厚度" 改为 "名义厚度"(2011版术语混淆)
```
