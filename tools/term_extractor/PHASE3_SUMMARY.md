## ✅ 任务1: diff命令

### 实现成果

**文件**: `tools/term_extractor/src/diff.rs` (261行)

**功能**:
- 比对两个param_mapping.json的差异
- 识别新增/删除/修改的术语
- 生成详细的差异报告(Markdown格式)
- 统计变更数量和类型

### 核心特性

1. **差异识别**
   - 新增术语: 检测新添加的term_id
   - 删除术语: 检测被移除的term_id
   - 修改术语: 检测字段变更

2. **详细比对**
   - 标准术语变更
   - 定义变更
   - 来源条款变更
   - SPEC参数名变更
   - 单位变更

3. **报告生成**
   - 统计摘要
   - 分类展示(新增/删除/修改)
   - 变更详情
   - 未变更列表

### 测试示例

**命令**:
```bash
term_extractor diff \
  --old rules/library/GB150/param_mapping.json \
  --new rules/library/GB150/param_mapping.json \
  --output docs/术语变更报告.md
```

**输出**: `docs/术语变更报告.md`

```markdown
# 术语变更比对报告

**生成时间**: 2025-06-30 08:29:49

## 📊 变更统计

- **新增术语**: 0 个
- **删除术语**: 0 个
- **修改术语**: 0 个
- **未变更**: 28 个
- **总变更数**: 0 处

## ✅ 未变更术语

以下术语保持不变:

- `pressure.design`
- `pressure.operating`
- ...
```

## ✅ 任务2: validate命令

### 实现成果

**文件**: `tools/term_extractor/src/validate.rs` (366行)

**功能**:
- 校验param_mapping.json的内部一致性
- 校验规则YAML的格式正确性
- 交叉校验参数引用
- 生成校验报告

### 核心特性

1. **param_mapping.json校验**
   - term_id唯一性检查
   - 参数名唯一性检查
   - 单位一致性检查
   - 引用完整性检查

2. **规则YAML校验**
   - 规则ID存在性检查
   - 规则格式正确性检查
   - 必填字段检查

3. **交叉校验**
   - 规则中引用的参数是否存在
   - 参数类型一致性
   - 单位匹配性

4. **问题分级**
   - Error: 必须修复的严重问题
   - Warning: 建议修复的警告
   - Info: 信息提示

### 测试示例

**命令**:
```bash
term_extractor validate \
  --rules rules/library/GB150/param_mapping.json \
  --rules rules/library/GB150.3/ \
  --report docs/校验报告.md \
  --check-only
```

**输出**: `docs/校验报告.md`

```markdown
# 校验报告

**生成时间**: 2025-06-30 08:31:39

## 📊 校验统计

- **术语总数**: 28
- **规则总数**: 5
- **错误**: 0 个
- **警告**: 11 个
- **信息**: 0 个

**状态**: ✅ 校验通过

## ⚠️ 警告

- **引用**: 术语 'pressure.design' 引用了不存在的术语 'pressure.calc' (位置: "param_mapping.json")
- **单位**: 参数 'sigma_allow' 缺少单位 (位置: "param_mapping.json")
...
```
