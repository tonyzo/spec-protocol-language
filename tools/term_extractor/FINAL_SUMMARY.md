- 生成YAML骨架文件

### 模板示例

**输入**: param_mapping.json
```json
{
  "term_id": "stress.allow",
  "spec_mapping": {
    "name": "sigma_allow"
  }
}
```

**输出**: YAML规则骨架
```yaml
- id: GB150.3-5.3.sigma
  clause: "5.3 / 公式 5-5"
  severity: error
  applicability:
    type: eq
    value:
      - {attr: element_type}
      - "内压圆筒"
  assertion:
    type: le
    value:
      - {param: sigma}          # 自动映射
      - {limit: sigma_allow}    # 自动映射
  message: "环向应力 {actual} MPa 超过许用应力 {expected} MPa"
```

## 💡 经验总结

### 成功经验

1. **数据结构设计合理**
   - param_mapping.json格式通用
   - 支持不同标准、不同术语类型
   - 易于扩展和维护

2. **文档生成工具可靠**
   - 自动生成结构化文档
   - 减少人工编写工作量
   - 保证格式一致性

3. **术语分类方法准确**
   - parameter/limit/attr三分法清晰
   - 判断口诀实用("计算出来的值→parameter")
   - 易于理解和执行

### 改进建议

1. **JSON不支持注释**
   - 初期添加了//注释导致语法错误
   - 建议: 使用JSON5或YAML格式(支持注释)
   - 或: 在外部文档中说明分类

2. **术语提取需人工校对**
   - 自动提取的术语置信度不一
   - 建议: 提供交互式校对界面
   - 或: 标记低置信度术语

3. **跨标准引用需明确**
   - 不同标准对同一概念的术语可能不同
   - 建议: 建立标准术语对照表
   - 或: 使用synonyms字段标注
